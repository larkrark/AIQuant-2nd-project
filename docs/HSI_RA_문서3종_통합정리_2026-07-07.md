## 1. ⭐ 최종 방법론 정정 (팀 코멘트) — 가장 중요

**최종 전략의 핵심은 Score 최대화가 아니라, HSI 상태별 목표비중으로 이동하는 "속도(λ)"를 조절하는 것.**
채택 판정도 종합점수식이 아니라 **사전 등록된 4개 조건을 모두 통과하는지**로 바뀌었다.

### 1.1 핵심 계산식

| # | 식 | 설명 |
|---|---|---|
| ① 비중 이동 | `actual_weight_t = previous_weight + λ_t × (target_weight_t − previous_weight)` | λ로 목표비중 반영 속도 조절 |
| ② Turnover | `Turnover = Σ|변화한 비중| / 2` | 회전율 |
| ③ 비용차감 수익률 | `net = gross − Turnover × 비용률` (10bp=0.001 → `net10bp = gross − turnover×0.001`) | 슬리피지·실거래비용 직접 추정 아님, 민감도용 |

### 1.2 동적 λ 변형

- **dynamic_v1**: 하나의 λ 고정이 아니라 **월별 조건에 따라 λ를 변경**(보고서 규칙).
- **dynamic_v1_macro**: dynamic_v1 + **MacroRisk 조건** 추가.
  - `MacroRisk = rate_up_flag + fx_up_flag ≥ 2` (금리 상승 압력 + 환율 상승 압력이 **동시**에 관찰된 달 = 고위험).

### 1.3 최종 채택 = "Score"가 아니라 "비열등 조건" (34_adoption_decision.py)

Score(λ)=Calmar + a·Sharpe − b·Turnover − c·CostSensitivity **는 쓰지 않음.**
대신 **OOS 10bp 비용차감 기준**으로 사전 등록된 4개 조건을 모두 통과하는지 판정:

```
① Calmar_net10bp ≥ (대칭 λ 후보 중 최우수 Calmar) × 0.90
② MDD가 대칭 λ=0.1 대비 2.0%p 이상 악화되지 않을 것
③ tail-month 평균수익이 대칭 λ=0.1 대비 0.3%p 이상 악화되지 않을 것
④ 평균 Turnover가 대칭 λ=0.3 평균 Turnover × 1.5 이하일 것

candidate_pass = (①) and (②) and (③) and (④)
```

> 즉 가중치 종합점수 → **사전등록 4조건 AND 통과**(비열등성)로 전환. 과최적화·튜닝 논란을 피하는 정직한 판정 구조.

## 4. 후속 실험 (비대칭·조건부 λ)

| 번호 | 실험 | 핵심 |
|---|---|---|
| E24 | 팩터로딩 진단 | 기존 λ 후보의 시장·채권·모멘텀·변동성·macro β + 36개월 rolling. "설명"이지 예측 아님 |
| E28 | 단일 λ 반응곡선 | λ 0→1 세밀 이동, 최고점 아닌 "안정 구간" 선정 |
| E29 | 비대칭 λ_up/λ_down grid | Δw*_069500<0(de-risking)→λ_down, ≥0(re-risking)→λ_up. 4×4=16셀, 대각선=대칭 |

## 5. 검증 게이트 8단계 (순서 고정)

① 재현·검산 → ② full-grid(필터 전 전량 공개) → ③ Turnover·비용 → ④ robustness(기간분할·상태별·tail-month·인접) → ⑤ IS/OOS → ⑥ walk-forward → ⑦ 누수 audit → ⑧ 과적합 점검.

- **IS/OOS**: IS 2012-04~2020-12(≈60%) 설계, OOS 2021-01~2026-06(≈40%) 고정 평가(threshold 재조정 금지).
- **Walk-forward**: 60개월 점검 → 12개월 평가 → 12개월 이동 반복.
- OOS가 강한 회복장 포함 → 재진입 속도(λ_up)에 유리 편향 가능 → 기간분할로 드러냄.

## 7. 우리 코드베이스와의 연결 (액션)

- 현재 `src/pipeline/stage_selection.py`는 **가중치 Score(`SELECTION_SCORE_WEIGHTS`)** 기반이다. 코멘트에 따르면 최종은 이를 쓰지 않고 **비열등 4조건(§1.3)**을 사용 → **`stage_selection`에 "비열등 판정" 함수(adoption_decision)를 추가**하는 것이 정합적(Score는 참고 지표로 강등 또는 병기).
- `stage_factor`(E24)·`stage_attribution`(§3.4)은 이미 구현·self-test 완료 → 요건(4)(5)에 직접 대응. 조원 데이터로 실행하면 본편 승격.
- `stage_d_lambda`의 dynamic λ 프리미티브(§1.2)는 dynamic_v1/macro 규칙(고위험→0.1, 안정완화→0.5, MacroRisk≥2)을 그대로 담을 수 있음 → 규칙·threshold를 config로 옮겨 IS에서만 결정.
- 상태별 목표비중 w*(§2)은 `stage_b_hsi.baseline_target_weights` 스텁을 채우는 정확한 규칙 → 바로 구현 가능.

## 8. 한 줄 요약

**최종 HSI RA는 "λ로 목표비중 반영 속도를 조절"하는 방어형 DAA이며, 후보 채택은 종합점수가 아니라 OOS 10bp 비용차감 하의 사전등록 4개 비열등 조건을 모두 통과하는지로 판정한다.** 알파는 수익이 아니라 낙폭 통제 엣지로 정의하고, 팩터로딩·attribution을 본편에 넣어 강의 요건을 채운다.
