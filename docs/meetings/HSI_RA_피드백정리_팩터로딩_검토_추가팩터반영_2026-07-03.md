## 문서 목적

발표/검토 과정에서 지적받은 세 가지 질문을 사실에 맞게 정리하고, 현재 프로젝트의 답변 범위와 후속 보강 계획을 분리한다.
핵심 원칙은 "과장하지 않고, 현재 수행한 검증과 아직 수행하지 않은 보강 과제를 명확히 구분한다"이다.

## 1. 질문 요약

표면적으로는 세 문장이지만, 실제로는 **과최적화, 검증 엄밀성, 팩터 기반 확장 가능성**을 묻는 질문이다.

| 질문 | 질문자가 확인하려는 것 | 현재 답변 방향 |
|---|---|---|
| Out-of-sample / In-sample | 전체 기간을 다 보고 후보를 고른 과최적화가 아닌지. IS/OOS 구분이 있는지. | 엄밀한 독립 OOS·walk-forward는 미수행. 다만 월말 신호→다음 달 수익률 적용, 기간분할 robustness, 큰 손실월, 비용 민감도는 수행. |
| Optimizer 여부 | λ를 수학적 최적화로 찾았는지, 단순히 좋은 값만 고른 것인지. | scipy optimizer식 단일 최적화는 아님. λ grid/sensitivity 실험 후 다지표 기준으로 후보 선별. |
| 팩터 넣어서 λ 개선 여지 | 시장·거시·변동성 팩터로 λ를 동적으로 바꿀 수 있는지. | 가능성 있음. 현재는 고정 λ 후보 검증 단계. 팩터 로딩·동적 λ는 후속 보강 과제. |

## 2. 질문 1 — Out-of-sample / In-sample 구분

- 월말에 관측 가능한 HSI 신호를 다음 달 ETF 수익률에 적용(같은 달 수익률 선반영 회피).
- 기간분할 robustness, HSI 상태별 진단, 큰 손실월 진단, 거래비용 민감도는 수행.
- 그러나 엄밀한 독립 holdout·walk-forward 검증까지 완료했다고 말하면 안 된다.
- 따라서 현재 결과는 "완전한 OOS 검증 완료"가 아니라 **시점 정합성과 기본 robustness를 갖춘 prototype 검증 결과**로 표현한다.

## 3. 질문 2 — Optimizer 여부

- scipy optimizer, Bayesian optimization, gradient search 같은 자동 optimizer는 미사용.
- λ 후보를 여러 값으로 나누어 sensitivity 실험 수행.
- 최고 CAGR 하나가 아니라 CAGR·MDD·Sharpe·Calmar·Turnover·비용 민감도·해석 가능성을 함께 고려.
- 따라서 λ=0.3은 "최적값"이 아니라 **"균형형 후보"**, λ=0.1은 **"저회전·보수형 후보"**로 표현.

**후속 보강 목적함수 예시**

```text
Score(λ) = Calmar(λ) + a × Sharpe(λ) − b × Turnover(λ) − c × CostSensitivity(λ)
```

단, 이런 최적화는 반드시 in-sample에서 후보를 고르고 out-of-sample에서 검증하는 구조로 설계한다.

## 4. 질문 3 — 팩터로 λ를 개선할 수 있는지

- 현재 최종 후보는 고정 λ=0.1, λ=0.3.
- macro companion·event balance는 보조 진단 layer로만 검토, λ를 팩터로 동적 조절하는 모델은 미구현.
- RA 확장 관점에서 팩터 로딩 분석과 attribution이 현재 가장 큰 보강 지점.
- 팩터 기반 동적 λ는 가능하지만, 현재 결과로 주장하기보다 **후속 보강 과제**로 제시한다.

## 5. 통합 답변 스크립트

현재 프로젝트는 완전한 optimizer 기반 상용 RA라기보다 **교육용 RA prototype**에 가깝다.
첫째, 엄밀한 독립 OOS·walk-forward는 미수행이나 시점 정합성과 기간분할 robustness·비용 민감도로 후보를 점검했다.
둘째, λ는 optimizer 단일 최적값이 아니라 sensitivity 실험을 통한 운용 목적별 후보(균형형 0.3, 보수형 0.1)다.
셋째, 팩터로 λ를 동적으로 개선할 여지는 있으며, 후속으로 factor loading·attribution을 추가한다.

## 6. 팩터로딩 분석 설계

### 6.1 왜 팩터로딩이 필요한가

RA 개발 요구사항에는 팩터 분석·팩터 로딩 알고리즘·팩터 추이가 포함된다. 현재 HSI 프로젝트는 가격 기반 신호·상태분류는 갖췄으나, 전략수익률이 어떤 팩터 노출에서 나왔는지 회귀형으로 분해한 분석이 부족하다. 팩터로딩은 "전략이 실제로 무엇에 노출되어 있었는가"를 설명하기 위한 보강 작업이다.

### 6.2 기본 후보 팩터

| 팩터 | 후보 변수 | 해석 | λ 개선과의 연결 |
|---|---|---|---|
| 시장 팩터 | 069500 또는 KOSPI200 월수익률 | 주식시장 베타 노출 | 시장 추세가 안정적이면 λ↑ |
| 채권/듀레이션 | 114260 월수익률, 금리 변화 | 국채 방어자산 노출 | 금리 상승기 비중 조정 |
| 현금성 | 153130 월수익률 또는 단기금리 | 현금성 방어 노출 | 위험구간 λ↓·현금성 유지 |
| 모멘텀 | 최근 3·6·12개월 수익률 | 추세 지속 여부 | 완화 추세 반복 시 λ↑ 근거 |
| 변동성 | rolling volatility, drawdown | 시장 불안정도 | 변동성 상승 시 λ↓ |
| Macro | 금리, 환율, macro risk score | 외부 거시 위험 | macro risk 강하면 λ↓ |
| HSI 상태 | direction, intensity, conflict, state dummy | HSI 내부 상태 노출 | 상태별 λ 차등 |

### 6.3 기본 회귀식

```text
R_strategy,t − R_BM,t = α + β1 Market_t + β2 Bond_t + β3 Momentum_t + β4 Volatility_t + β5 MacroRisk_t + ε_t
```

β가 팩터 로딩이다. β가 크면 해당 전략이 그 팩터에 많이 노출되어 있었다는 뜻이다.

### 6.4 Rolling factor loading

전체 기간 단일 회귀는 시간에 따른 노출 변화를 못 본다. 36개월 rolling을 기본, 24개월은 부록 민감도로 둔다.

### 6.6 추가 검토 후보 팩터 (추가 반영)

기존 7팩터와 **겹치지 않으면서** ①경제적 근거가 분명하고 ②한국 데이터로 계산 가능하며 ③방어형 overlay·동적 λ에 직접 연결되고 ④룩어헤드를 피할 수 있는 확장 후보다.

**Tier 1 — 우선 추가 권장 (저비용·고가치·비중복)**

| 팩터 | 경제적 근거 | 계산(가용 데이터) | 방어/λ 연결 | 주의 |
|---|---|---|---|---|
| 주식-채권 상관 레짐 | 분산효과가 깨지는 구간(동반 하락) = 방어 실패 | 069500 vs 114260 36개월 rolling corr | 상관 (+)전환 시 λ↓·현금성↑ | 롤링 corr는 후행적 |
| 장단기 금리차(term spread) | 경기·침체 선행지표 | 국고 10Y−3Y, 또는 114260−153130 스프레드 | 축소/역전 시 방어 강화 | 월말 관측치만 사용 |
| 내재변동성(VKOSPI) 레벨·Δ | 시장의 예상 공포(선행), 실현변동성과 구분 | VKOSPI z-score·변화 | 급등 시 λ↓(과잉매매 방지) | 실현변동성과 다중공선성 점검 |
| 원/달러 환율 변화·변동성 | KRW 약세 = 외국인 이탈·리스크오프(한국 특수) | USD/KRW 월변화·rolling vol | 급등 시 방어자산 강화 | 국면별 방향 해석 상이 |
| 미국시장 스필오버(1개월 시차) | 한국 증시의 미국 연동·overnight lead | 전월 S&P500 수익률·전월 VIX | 미국 위험 상승 시 선제 방어 | 반드시 시차(lag) 적용해 룩어헤드 차단 |

**Tier 2 — 가치 있으나 데이터·처리 필요**

- 신용스프레드(회사채AA−−국고3Y): 위험선호 직접 지표, 확대 시 방어.
- 외국인 순매수 플로우(KRX): 한국 주식 최강 드라이버 중 하나(발표 시차 주의).
- 거래대금/유동성: 시장 참여·쏠림, 급감은 스트레스 신호.
- 원자재(유가·구리): 글로벌 성장·인플레, 수입의존 한국에 유효.
- 하방·꼬리위험(semi-deviation·CVaR·drawdown 깊이): 단순 변동성과 달리 비대칭 하방만 포착 → 방어형 목적에 정합.

**Tier 3 — 실험적·보조**

- 계절성(turn-of-month·월 더미·sell-in-May): 저렴하나 효과 약함, 부록용.
- 팩터 가속도(Δfactor, 2차 미분): 레벨이 아닌 변화 속도 → 동적 λ에 직접 유용.
- 레짐 상호작용(팩터 × HSI 상태 더미): 조건부 로딩 → 동적 λ 근거 강화.
- 기준금리 사이클 더미(인상/동결/인하): 듀레이션 포지셔닝.

### 6.7 팩터 선정 방법론 주의 (추가 반영)

1. **표본이 작다(172개월, 36m 롤링) → 팩터를 많이 넣으면 과최적화.** 직교(비중복) 팩터 5~6개로 절제하거나, 여러 위험팩터를 PCA/합성 risk score 1개로 묶는다. VIF로 다중공선성 점검.
2. **PIT·발표 시차:** CPI·외국인플로우·GDP는 공표 지연이 있어 그대로 쓰면 룩어헤드. GDP를 뺀 논리와 동일하게 적절히 시차를 주거나 제외.
3. **표준화도 룩어헤드 없이:** z-score는 전체표본이 아니라 expanding/rolling window로 계산.
4. **기존 7팩터와 상관 점검:** VKOSPI↔실현변동성, 환율↔macro 등 일부 겹침 → 상관표를 먼저 그려 중복 제거 후 채택.

**1차 추천:** Tier 1 다섯 개(특히 주식-채권 상관 레짐, term spread, VKOSPI, 미국 스필오버)를 넣고, `stage_factor.py` 회귀에서 상관·VIF로 3~4개로 확정. 하방위험(semi-dev)은 방어형 스토리에 잘 맞아 Tier 2 중 우선 추가.

## 7. 팩터 기반 동적 λ 개선 아이디어

핵심은 HSI 상태 자체를 바꾸는 게 아니라, **HSI 목표비중이 실제 포트폴리오에 반영되는 속도(λ)를 조절**하는 것이다.

| 시장 조건 | 동적 λ 방향 | 이유 |
|---|---|---|
| 변동성 상승·drawdown 확대 | λ 낮춤 | 신호 급변 시 과잉매매 축소, 방어적 이동 |
| 위험 완화 상태 수개월 지속 | λ 높임 | 완화 신호가 노이즈 아닐 가능성 ↑ |
| 금리·환율 macro risk 동시 상승 | λ 낮춤/방어자산 강화 | 주식·채권 위험 동시 확대 가능 |
| 모멘텀 + HSI relief 동시 확인 | λ 높임 | 상태분류·추세 동일 방향 |

**규칙 기반 예시**

```text
기본 λ = 0.3
고위험: volatility_z > 1 또는 drawdown < −10% → λ = 0.1
안정 완화: risk_relief 3개월 지속 & volatility_z < 0 → λ = 0.5
```

**스코어 기반 예시**

```text
λ_t = clip(λ_min, λ_max, λ0 + a × TrendScore_t − b × VolRisk_t − c × MacroRisk_t)
```

복잡도가 높아지므로 단순 λ=0.1/0.3 대비 실제 개선 여부를 별도 검증(train/validation/test 또는 walk-forward)해야 한다.

## 8. 후속 실험 계획 및 우선순위

1. `stage_factor.py` — 전략별 월수익률 + 팩터 후보 병합, λ=0.1/0.3·baseline·Fixed BM·EW 전체기간 회귀.
2. Rolling factor loading — 36개월 rolling regression으로 노출 변화 시계열.
3. `stage_attribution.py` — BM 대비 초과수익을 자산배분·HSI timing·λ smoothing·비용 효과로 단순 분해.
4. 동적 λ 예비실험 — 규칙 기반 dynamic lambda 1개 설계 후 고정 λ와 비교.
5. 검증 분리 — dynamic lambda는 train/validation/test 또는 walk-forward로 평가.

| 우선순위 | 작업 | 이유 |
|---|---|---|
| 1 | 팩터로딩 회귀 표 | RA 요구사항의 팩터 분석 갭을 가장 직접 메움 |
| 2 | 간단한 attribution 표 | 성과가 어디서 나왔는지 설명 |
| 3 | 동적 λ 아이디어 검토 | 후속 개선 가능성 제시 |
| 4 | walk-forward 설계 | OOS 지적에 대한 가장 강한 보강 |

## 9. 제출용 결론

이번 지적은 프로젝트의 한계를 확인하는 중요한 질문이었다. 현재 HSI RA는 시점 정합성·기간분할 robustness·비용 민감도 등 기본 검증은 수행했으나, 독립 OOS·walk-forward까지 완료한 것은 아니다. λ는 optimizer 단일 최적값이 아니라 sensitivity 실험을 통한 운용 목적별 후보다. 팩터 기반 동적 λ 가능성은 존재하나, 고정 λ 후보의 방어성을 먼저 검토했고 factor loading·attribution·dynamic lambda·walk-forward는 후속 보강 과제로 남긴다.
