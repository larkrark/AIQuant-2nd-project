# HSI 목표비중 셔플 대조군 실험 — 논의사항 정리

- 정리일: 2026-07-08 (팀 채팅 오후 4시대 논의)
- 참여: 팀원(제안), 권보성(실행), 김근형(설계 보완)
- 배경: ablation 실험(§섹션C)에서 "HSI 방향이 낙폭 통제의 주동력(F−E ≈ +4.6%p)"이라는 결론이 나왔으나, HSI 방향이 **우연이 아닌 실제 정보**인지 통계적으로 검증할 대조군이 없다는 지적.

---

## 1. 제안된 실험 설계 (원안)

λ를 정하는 변동성·drawdown 규칙은 **그대로 유지**하고, HSI가 주는 **목표비중만 무작위로 셔플**하여 백테스트한다.
예: 원래 "위험 경고" 달에 위험자산 20%를 배정했다면, 셔플 버전에서는 그 자리에 임의 상태의 목표비중을 무작위 배정.
→ "HSI 방향은 랜덤, λ 조절만 살아있는" 대조군이 만들어진다.

**판정 기준:**

| 결과 | 해석 |
|---|---|
| 셔플 ≈ dynamic_v1 성능 유사 | HSI 방향 기여 낮음. 성과는 변동성 기반 λ 조절의 산물 (**HSI 무용**) |
| 셔플 시 성능 붕괴 | HSI 방향이 실제 유효한 정보였다는 증거 (**HSI 유효**) |

제안자 본인도 시행 가능성이 낮거나 더 나은 방법이 있으면 다른 접근도 좋다고 열어둠.

## 2. 개념 정정 (김근형)

λ는 **방향 신호가 아니라** 이전 비중 → 목표비중 사이의 **이동속도(상대속도) 개념**이다.
"변동성·drawdown 규칙이 방향을 정한다"는 표현은 부정확하며, 방향(목표비중)은 HSI가, 속도(λ)는 변동성·drawdown이 담당한다.
또한 "낙폭이 −12%까지 줄어든다"는 가정 자체가 검증 대상이므로 이 대조군 제안은 타당하다고 동의.

## 3. 설계 보완 사항 (합의된 방향)

**(1) 1회 셔플로는 부족 → 몬테카를로 다회 반복.** 셔플을 여러 번 반복해 성과 분포를 만들고, 실제 dynamic_v1이 그 분포의 어디에 위치하는지(백분위/유의성)로 판정한다.

**(2) 매달 독립 셔플은 비현실적 → 블록 셔플.** 매달을 독립적으로 섞으면 원본 HSI가 만든 적 없는 "매달 비중이 요동치는" 인위적 경로가 생긴다. 상태 지속성 구조를 유지하기 위해 **3~6개월 블록 단위**로 끊어 블록의 배치 시점을 무작위화한다.

## 4. 기존 분해 지표와의 관계 (이론적 근거)

추가실험으로 **exposure_effect / timing_effect** 분해가 이미 산출되어 있음.

- **exposure_effect** = 시간평균 비중 차이. 목표비중을 무작위로 섞어도 상태별 비중벡터 집합의 재배열이므로 시간평균이 거의 보존 → **셔플에 불변**.
- **timing_effect** = 그 비중을 "언제" 썼는지의 시점 정보. 셔플이 정확히 이 시점 정보를 파괴 → **셔플 테스트는 사실상 timing_effect(+25.80%p, FULL 기준)가 우연인지에 대한 검정과 동치**.

예측: 실제 timing_effect가 이미 강한 양수이므로, 셔플 분포 대비 dynamic_v1이 우위로 나올 것으로 예상. 다만 안전하게 검증하고 가는 것이 중요하다는 데 동의.

## 5. 액션 아이템

- [ ] **권보성**: 제안된 방법(블록 셔플 몬테카를로)으로 대조군 비교 백테스트 및 시각화
- [ ] **팀**: 김근형의 블록 셔플 설계(블록 배치 무작위화가 timing 정보만 파괴하고 exposure는 보존)에 대한 의견 회신

## 6. 미결 설계 결정 사항 (실행 전 확정 필요)

1. **블록 길이**: 3개월 vs 6개월, 또는 둘 다 (민감도 확인용)
2. **반복 횟수**: 몬테카를로 시행 수 (예: 500~1,000회)
3. **판정 지표**: MDD / Calmar / timing_effect 중 무엇을 1차 기준으로 볼지, 백분위 임계(예: 실제값이 셔플 분포 상위 5% 밖이면 유효)
4. **블록 경계 처리**: 기간이 블록 길이로 나누어떨어지지 않을 때 잔여 구간 처리 방식
5. **셔플 범위**: 목표비중 벡터 전체를 블록으로 섞을지, 상태 라벨을 섞고 비중 매핑을 다시 적용할지 (둘은 동치가 아님 — 확인 필요)
