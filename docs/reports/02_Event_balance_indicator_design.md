# 02_Event_balance_indicator_design

## 실험명
**02번 HSI 내부 사건균형지표 설계**

## 1. 목적

02번 단계는 HSI 입력 신호의 과거 분포를 기준으로 위험 사건과 완화 사건을 정의하는 내부 보조지표 설계 단계이다. 이 지표는 외부 뉴스나 사건 달력이 아니다. HSI 입력 신호 자체에서 위험 악화 방향 또는 위험 완화 방향의 극단 신호가 최근 기간에 얼마나 반복되었는지를 확인한다.

## 2. 설계 원리

02번의 기본 아이디어는 다음과 같다.

```text
원신호
→ HSI 위험 방향으로 부호 통일
→ 과거 rolling 분포의 20/80 분위수 기준 계산
→ 위험 사건 / 완화 사건 flag 생성
→ 1개월·3개월·6개월·12개월 창에서 사건 반복 정도 요약
→ event_balance, event_intensity 생성
```

분위수(설명: 데이터를 작은 값부터 큰 값까지 정렬했을 때 특정 위치에 해당하는 값이다. 20분위수는 하위 20%, 80분위수는 상위 20%의 경계로 볼 수 있다.)  
event_balance(설명: 위험 사건과 완화 사건 중 어느 쪽이 더 우세한지 보는 내부 보조지표이다.)  
event_intensity(설명: 위험 또는 완화 사건이 얼마나 강하게 반복되었는지 보는 내부 보조지표이다.)

## 3. 신호별 방향 통일표

| 신호 | 신호군 | 방향 부호 | 사건 해석 | bundle 포함 |
| --- | --- | --- | --- | --- |
| ret_1m | return | -1 | 수익률이 높을수록 위험 완화이므로 부호를 반전한다. | 1 |
| ret_3m | return | -1 | 수익률이 높을수록 위험 완화이므로 부호를 반전한다. | 1 |
| ma_gap | trend | -1 | 이동평균 대비 위치가 좋을수록 위험 완화로 본다. | 1 |
| momentum | trend | -1 | 모멘텀이 좋을수록 위험 완화로 본다. | 1 |

## 4. 이 단계의 위치

02번 사건균형지표는 최종 ETF 비중을 직접 결정하지 않는다. 이후 08번 정합성 진단과 09번 보조 필터 백테스트에서 “HSI 상태와 내부 사건 반복 정도가 서로 맞는지”를 확인하는 데 사용된다.

따라서 02번은 최종 후보 전략이 아니라, HSI 신호 해석을 보완하기 위한 진단 계층이다.

## 5. 결론

02번 단계는 외부 사건을 추가하는 단계가 아니라, HSI 내부 신호가 극단적으로 반복되는지를 확인하는 보조 설계이다. 최종 전략의 중심은 여전히 HSI 상태분류와 ETF 비중 전환이며, event balance는 이를 설명하거나 검증하기 위한 보조 자료로 사용한다.
