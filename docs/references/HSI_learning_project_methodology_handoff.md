# HSI 프로젝트 학습자료 연결·실험설계·보고서 논리 정리

## 컴퓨팅 수업 복습 내용을 HSI 기반 ETF 방어형 자산배분 프로젝트 방법론으로 연결하기

작성 목적: 지금까지 정리한 컴퓨팅 수업자료, 주식·시계열 학습 코드, HSI 프로젝트 실험 설계, 금장친 피드백을 하나의 전달 문서로 묶는다.

## 1. 핵심 결론

- 컴퓨팅 수업의 날짜 인덱스 처리, 리샘플링, rolling 이동평균은 HSI의 월말 가격, 월간 수익률, ma_gap, momentum 생성과 직접 연결된다.

- ADF, 차분, ACF/PACF는 가격 수준 자체보다 변환 신호를 쓰는 이유를 설명한다.

- 과적합 복습은 θ 민감도와 Grid Search에서 단일 1등 조합만 선택하면 안 되는 근거가 된다.

## 2. 프로젝트 핵심 정의

본 프로젝트는 HSI로 미래 수익률을 직접 예측하는 것이 아니라, 가격 기반 신호를 시장상태로 번역하고, 그 상태를 ETF 비중 조정으로 연결했을 때 방어형 자산배분 전략으로서 의미가 있는지 검증한다.

## 3. 학습자료와 프로젝트 연결 지도

| 학습 내용 | HSI 프로젝트 연결 |
|---|---|
| datetime/resample/rolling | 월말 가격, 월간 수익률, ma_gap, momentum |
| 분포·IQR·box plot | 월간 수익률 꼬리위험, 위기월 확인 |
| 상관·회귀선 | HSI 신호와 다음 달 수익률의 관계 진단 |
| ADF·차분·ACF/PACF | 가격 수준 대신 변환 신호 사용 근거 |
| 과적합·Grid Search | θ/Grid Search에서 안정성 기준 사용 |
| PCA | 확장 신호 중복 점검 |

### 학습자료 연결 문장
본 프로젝트는 금융 이론뿐 아니라 데이터 처리와 시계열 분석의 기본 절차를 함께 반영하였다. 수업에서 학습한 날짜 인덱스 처리, 리샘플링, 이동평균, 분포 확인, 상관·회귀선 해석, 과적합 검토 방법을 HSI 입력 신호 점검과 백테스트 해석에 연결하였다. 특히 HSI는 가격 수준을 직접 예측하기보다, 수익률·이동평균 괴리·변동성·상대강도와 같은 변환 신호를 통해 시장상태를 분류하는 구조로 설계하였다.

### 예측 과장 방지 문장
HSI는 다음 달 수익률을 직접 맞히는 예측모델이 아니라, 현재 관측 가능한 가격 기반 신호를 위험 악화와 위험 완화 방향으로 정렬하여 시장상태를 분류하는 보조지표이다. 따라서 본 프로젝트의 평가는 예측 정확도보다 상태분류의 해석 가능성, 손실 방어력, 회전율, 거래비용 반영 후 안정성에 초점을 둔다.

### 과적합 방지 문장
θ 민감도 실험과 비중 규칙 탐색에서는 CAGR이 가장 높은 조합을 단순 선택하지 않는다. 수업의 다항회귀 실습에서 확인했듯이, 복잡도를 높이면 특정 데이터에만 과도하게 맞는 결과가 나타날 수 있다. 본 프로젝트에서는 MDD, Turnover, Sharpe, Calmar, 상태분포 안정성을 함께 확인하여 특정 파라미터 하나에만 의존하는 결과를 경계한다.

### 시장 사건 달력 문장
시장 사건 달력은 HSI 계산이나 포트폴리오 비중 결정에 직접 사용하지 않는다. HSI 상태와 백테스트 결과를 먼저 산출한 뒤, 주요 시장 사건 구간과 사후적으로 대조하여 해석 가능성과 위기구간 설명력을 확인하는 보조 자료로 사용한다.

### 금장친 피드백 반영 문장
학습자료 기반 정리의 가치는 새로운 금융 모델을 추가로 제안하는 데 있지 않다. 그보다 컴퓨팅 수업에서 배운 시계열 처리, 분포 확인, 상관·회귀, 과적합 방지, 시각화 방법을 HSI 프로젝트의 데이터 처리와 보고서 논리로 연결해 주는 데 있다.
