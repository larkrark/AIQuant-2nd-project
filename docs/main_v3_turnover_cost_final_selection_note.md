# main_v3 Turnover, 거래비용, 최종 후보 판단 기준

- 작성 목적: HSI 5상태 overlay 후속 실험에서 Turnover 필터, 거래비용 단순화 가정, 최종 판단표 구조를 팀 기준으로 고정한다.
- 적용 범위: 1차 Grid Search 결과 이후의 2차 후보 선별, 거래비용 민감도 검증, 최종 후보 판단표 작성.

## 1. Turnover 해석

Turnover는 리밸런싱 시점에 자산 비중이 얼마나 바뀌었는지를 나타내는 값이다. Turnover가 클수록 매매가 많이 일어난 것으로 해석할 수 있으므로, 성과가 좋아 보여도 과도한 매매가 필요한 조합은 후순위로 둔다.

본 프로젝트에서는 Turnover를 수익률 지표와 같은 최종 성과 지표가 아니라, 실전 가능성이 낮은 후보를 걸러내는 필터로 사용한다.

## 2. 2차 후보 Turnover 필터

1차 결과표는 Turnover 상한을 적용하기 전의 전체 Grid Search 결과로 둔다. 이후 2차 후보표에서는 아래 기준을 적용해 과도한 매매 후보를 제거한다.

| 구분 | 기준 | 역할 |
|---|---:|---|
| 평균 Turnover | `avg_turnover <= 0.05` | 평상시 리밸런싱 부담이 과도한 후보 제거 |
| 최대 Turnover | `max_turnover <= 0.25` | 특정 월에 비중이 급격히 바뀌는 후보 제거 |

기준은 예비값이며, 후보가 지나치게 적거나 많으면 팀 합의로 조정할 수 있다. 다만 최종 보고서에서는 기준을 사후적으로 바꾼 것처럼 보이지 않도록, 조정 사유를 함께 남긴다.

## 3. 거래비용 단순화 가정

거래비용은 실제 ETF별 수수료, 스프레드, 세금, 체결 오차를 정밀 추정하는 방식이 아니라, 백테스트 결과의 민감도를 확인하기 위한 단순화 가정으로 둔다.

계산식은 다음과 같이 통일한다.

```text
월별 거래비용 = 월별 Turnover * 거래비용률
비용 반영 월수익률 = 기존 월수익률 - 월별 거래비용
```

예를 들어 어떤 달의 Turnover가 `0.20`이고 거래비용률이 `0.10%`, 즉 `0.001`이면 월별 거래비용은 `0.20 * 0.001 = 0.0002`이다. 이는 해당 월 수익률에서 `0.02%p`를 차감하는 방식으로 반영한다.

## 4. 거래비용률 시나리오

| 비용률 | 해석 | 사용 목적 |
|---:|---|---|
| 0.00% | 비용 미반영 기준 | 기존 결과표와 비교하는 기준선 |
| 0.05% | 낮은 비용 가정 | 낮은 거래마찰 환경에서의 민감도 확인 |
| 0.10% | 보통 비용 가정 | 기본 보고용 비용 반영 시나리오 |
| 0.20% | 보수적 비용 가정 | 비용이 커져도 후보가 유지되는지 확인 |

최종 판단에서는 `0.10%`를 중심 시나리오로 보고, `0.20%`에서 성과가 크게 무너지는 후보는 후순위로 둔다.

## 5. 1차, 2차, 최종 판단표 흐름

| 단계 | 표 | 목적 |
|---|---|---|
| 1차 | 전체 Grid Search 결과표 | 비용과 Turnover 필터 적용 전 넓은 후보군 확인 |
| 2차 | Turnover 상한 적용 후보표 | 평균/최대 Turnover 기준을 통과한 후보만 유지 |
| 3차 | 거래비용 민감도 표 | 비용률 0.00%, 0.05%, 0.10%, 0.20% 적용 후 성과 안정성 확인 |
| 최종 | 최종 판단표 | MDD, Calmar, Sharpe, Turnover, Robustness까지 종합 판단 |

## 6. 최종 판단표 필수 컬럼

최종 판단표는 수익률 1등을 뽑는 표가 아니라, 해석 가능하고 방어 목적에 맞으며 비용과 검증을 통과한 후보를 고르는 표로 사용한다.

| 컬럼 | 설명 |
|---|---|
| `candidate_id` | 후보 조합 ID |
| `signal_set` | 사용한 HSI 입력 신호 조합 |
| `weight_rule_id` | 비중 규칙 또는 Grid 조합 ID |
| `method` | rank 또는 zscore |
| `cagr_after_cost_0bp` | 거래비용 0.00% 반영 CAGR |
| `cagr_after_cost_5bp` | 거래비용 0.05% 반영 CAGR |
| `cagr_after_cost_10bp` | 거래비용 0.10% 반영 CAGR |
| `cagr_after_cost_20bp` | 거래비용 0.20% 반영 CAGR |
| `mdd` | 최대낙폭 |
| `calmar` | CAGR / MDD 기반 방어 성과 |
| `sharpe` | 변동성 대비 성과 |
| `avg_turnover` | 평균 Turnover |
| `max_turnover` | 최대 Turnover |
| `turnover_filter_pass` | Turnover 기준 통과 여부 |
| `robustness_pass_count` | Robustness 검증 통과 개수 |
| `rank_zscore_consistency` | rank/zscore 결과 일관성 |
| `final_decision` | keep, review, drop |
| `decision_reason` | 최종 판단 사유 |

## 7. 팀 공유용 의견 문안

거래비용률 후보는 `0.00%`, `0.05%`, `0.10%`, `0.20%` 네 구간으로 두는 것이 적절해 보입니다. `0.00%`는 기존 결과와 비교하는 기준선으로 두고, `0.10%`를 기본 비용 반영 시나리오, `0.20%`를 보수적 민감도 검증으로 쓰면 보고서에서 설명하기 좋습니다.

계산은 말씀하신 방식대로 `월별 거래비용 = 월별 Turnover * 거래비용률`로 단순화하고, 월수익률에서 해당 비용을 차감하는 방식이 가장 깔끔합니다. 1차 Grid Search에서는 비용 미반영 성과로 후보를 넓게 보고, 2차에서는 평균 Turnover와 최대 Turnover 상한을 적용해 매매가 과도한 후보를 제거한 뒤, 최종 판단표에서 비용 반영 후 CAGR, MDD, Sharpe, Calmar, Turnover, Robustness를 같이 비교하는 흐름을 제안드립니다.
