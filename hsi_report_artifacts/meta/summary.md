## 1. HSI 개념과 실험 의도

본 프로젝트의 HSI는 미래 수익률을 직접 예측하는 모델이 아니라, 가격 기반 신호를 이용해 ETF 시장상태를 해석하고 위험자산 비중 조절을 보조하기 위한 내부 합성지표이다. 여기서 HSI는 기존 금융시장에서 널리 통용되는 표준 지표가 아니라, 본 프로젝트에서 정의한 Hourglass Signal Index이다.

본 프로젝트의 실험 목적은 “가장 높은 수익률을 낸 조합”을 찾는 것이 아니다. 핵심 목적은 HSI 상태분류가 실제 ETF 데이터에서 설명 가능하게 생성되는지, 그 상태분류가 ETF 비중 조정 규칙과 연결될 수 있는지, 그리고 위험 완화와 Turnover 관리 측면에서 운용 가능한 overlay 구조로 발전할 수 있는지를 확인하는 것이다.

따라서 각 실험은 하나의 연구질문을 맡는다. 어떤 실험은 데이터 단위를 점검하고, 어떤 실험은 HSI 상태분류를 만들며, 어떤 실험은 사건균형지표나 상대속도처럼 HSI 내부 흐름을 진단한다. 이후 baseline, λ 부분조정, θ 민감도, macro companion layer를 통해 어떤 조합을 최종 후보로 가져갈지 판단한다.

본 프로젝트의 결론은 “HSI가 수익률을 맞힌다”가 아니라, “HSI 상태분류는 비중 전환 속도와 보조 진단 레이어를 함께 설계할 때 ETF overlay 전략으로 검토할 가치가 있다”이다.

## 2. 전체 실험 흐름 요약

본 프로젝트는 다음 순서로 진행하였다.

1. ETF 유니버스와 월간 수익률 데이터를 정리한다.
2. 가격 기반 HSI 입력 신호를 계산한다.
3. HSI direction, intensity, conflict 조건을 이용해 5상태를 만든다.
4. HSI 5상태를 ETF 목표 비중으로 연결해 baseline 백테스트를 수행한다.
5. baseline의 한계를 확인한 뒤, λ 부분조정으로 비중 전환 속도를 조절한다.
6. 사건균형지표와 상대속도 지표를 이용해 HSI 내부 신호 흐름을 진단한다.
7. θ 민감도와 거래비용 민감도를 확인해 특정 파라미터에 과도하게 의존하지 않는지 점검한다.
8. 금리, 환율, GDP를 이용한 macro companion layer를 HSI 보조 해석 레이어로 검토한다.
9. 최종 후보는 CAGR만으로 고르지 않고, MDD, Turnover, 비용 민감도, 해석 가능성을 함께 보고 판단한다.

초기 prototype 실험은 삭제하지 않고 보존하지만, 최종 발표와 보고서에서는 main_final 기준으로 재구성한 파이프라인과 23번 보고서 후보표를 기준으로 사용한다.


## 3. 데이터 처리와 입력 자료 정리

데이터 처리 단계의 목적은 HSI 계산과 백테스트가 같은 기준 데이터 위에서 실행되도록 만드는 것이다. 이 단계에서는 ETF 유니버스, 자산군 분류, 월말 가격, 월간 수익률, HSI 입력 신호를 정리한다.

월간 수익률은 반드시 단위를 구분한다. 백테스트에는 decimal 단위를 사용하고, 원 파이프라인 확인에는 pct 단위를 따로 보존한다. 이 구분은 매우 중요하다. 과거 prototype 단계에서는 pct 수익률을 decimal처럼 사용하는 위험이 있었기 때문에, 최종 파이프라인에서는 수익률 단위를 명시적으로 점검한다.

00번은 프로젝트 설정을 고정하고, 01번은 데이터 번들을 최종 실험용 산출물로 변환한다. 02번은 HSI 입력 신호에서 사건균형지표를 만들고, 03번은 월말 신호와 다음 달 수익률을 연결한다. 이 과정을 통해 월말에 관측 가능한 신호를 다음 달 포트폴리오 수익률에 적용하는 시점 정합성을 유지한다.


## 4. HSI 지표 계산법 총정리

HSI는 여러 가격 기반 신호를 하나의 상태 해석 체계로 통합하기 위한 내부 합성지표이다. 각 입력 신호는 위험 악화 방향과 위험 완화 방향을 해석할 수 있도록 같은 방향성 기준으로 정리한다.

HSI 계산에서 중요한 개념은 direction, intensity, conflict이다. direction은 전체 신호가 위험 완화 쪽으로 기울었는지, 위험 악화 쪽으로 기울었는지를 나타낸다. intensity는 신호의 강도를 나타내며, conflict는 위험 완화 신호와 위험 악화 신호가 동시에 강하게 나타나는 혼합 구간을 설명한다.

상대속도 진단은 특정 신호가 HSI 전체 중심 흐름보다 빠르게 움직이는지 확인하기 위한 보조 분석이다. 예를 들어 지표 A와 지표 B가 모두 위험 악화 방향으로 움직이더라도, 어느 지표가 먼저 빠르게 반응했는지에 따라 상태 전환의 해석이 달라질 수 있다. 이 진단은 미래 수익률을 예측하기 위한 신호가 아니라, HSI 내부 신호의 반응 속도와 상태 변화의 배경을 설명하기 위한 도구이다.

사건균형지표는 HSI 입력 신호에서 위험 사건과 완화 사건의 누적 흐름을 비교하기 위한 지표이다. 과거 분위수 기준으로 위험 사건과 완화 사건을 정의하고, 위험 사건 비율에서 완화 사건 비율을 뺀 값을 사건균형으로 해석한다. 사건강도지표는 위험 사건과 완화 사건이 동시에 얼마나 많이 발생했는지를 나타낸다. 따라서 사건균형지표는 HSI 상태를 대체하는 신호가 아니라, HSI 상태가 어떤 내부 사건 흐름 위에서 만들어졌는지 설명하는 보조 진단지표이다.

| 개념 | 의미 | 사용 목적 | 전략 입력 여부 |
|---|---|---|---|
| HSI direction | 위험 완화 또는 위험 악화 방향 | 시장상태 방향 해석 | 사용 |
| HSI intensity | 상태 신호의 강도 | 상태 전환 강도 해석 | 사용 |
| conflict | 완화와 악화 신호의 충돌 | 혼합 구간 탐지 | 사용 |
| relative speed | 개별 신호의 상대 변화 속도 | 내부 반응속도 진단 | 직접 사용하지 않음 |
| event balance | 위험 사건과 완화 사건의 누적 균형 | HSI 상태 보조 해석 | 제한적 보조 필터 |
| event intensity | 사건 발생 강도 | 충돌·고강도 구간 해석 | 직접 사용하지 않음 |

## 5. HSI 5상태 분류와 baseline 전략

HSI 5상태 분류는 HSI를 포트폴리오 비중 조정에 사용할 수 있는 시장상태 언어로 바꾸는 단계이다. 상태는 `risk_relief`, `neutral_watch`, `conflict`, `risk_warning`, `accident_zone`으로 구성한다.

각 상태는 다음과 같이 해석한다.

| HSI 상태 | 해석 | 기본 방향 |
|---|---|---|
| risk_relief | 위험 완화 | 위험자산 비중 확대 가능 |
| neutral_watch | 중립 관찰 | 균형 유지 |
| conflict | 신호 충돌 | 보수적 관찰 |
| risk_warning | 위험 악화 | 위험자산 축소 |
| accident_zone | 사고 구간 | 강한 방어 |

baseline 전략은 이 5상태를 ETF 목표 비중으로 직접 연결한 기준선이다. 이 전략은 HSI 상태가 실제 ETF 비중 조정으로 연결될 수 있는지 확인하기 위한 실험이다.

실행 결과 baseline은 EW보다 CAGR과 최종 누적수익률은 높았지만, MDD와 연변동성, Turnover도 함께 커졌다. 따라서 baseline은 최종 방어형 전략으로 채택하지 않고, 이후 λ 부분조정 실험의 비교 기준으로 사용한다.


## 6. 실험 결과 판단 방법

본 프로젝트에서는 CAGR이 가장 높은 전략을 자동으로 채택하지 않는다. HSI overlay 후보는 다음 기준을 함께 본다.

1. EW 대비 수익성 또는 Calmar가 개선되는가?
2. baseline 즉시비중 전략보다 MDD가 완화되는가?
3. 평균 Turnover가 관리 가능한 수준인가?
4. 거래비용 민감도에서 성과가 크게 훼손되지 않는가?
5. 특정 지표나 특정 파라미터 하나에만 과도하게 의존하지 않는가?
6. HSI 연구목적과 맞게 해석 가능한가?
7. 발표와 보고서에서 설명 가능한 구조인가?

표나 그림에서 특정 결과가 튀는 경우에는 바로 채택하지 않는다. 먼저 해당 결과가 데이터 오류, 수익률 단위 오류, 중복 시계열, 과도한 Turnover, 특정 기간 효과 때문인지 확인한다. 설명 가능한 결과라면 해석을 붙이고, 설명이 어렵거나 운용 관점에서 부담이 크다면 후보에서 제외하거나 부록으로 보낸다.

## 7. 주요 실험 λ, event filter, θ, 비용 민감도 결론
baseline 즉시비중 전략은 HSI 상태를 ETF 비중으로 연결하는 기준선이다. 하지만 MDD와 Turnover 부담이 커서 최종 후보로 채택하지 않는다.

λ 부분조정 실험은 현재 가장 중요한 개선 결과이다. λ는 현재 비중에서 목표 비중으로 이동하는 속도를 조절한다. λ=1.0은 즉시 목표비중 이동이고, λ가 작아질수록 기존 비중을 더 많이 유지하며 천천히 이동한다. 실험 결과 λ를 낮추면 MDD와 Turnover가 함께 완화되는 경향이 관찰되었다.

현재 기준에서는 λ=0.3을 균형형 우선 검토 후보로 두고, λ=0.1을 방어형 저회전 후보로 둔다. 다만 λ=0.3을 최적값으로 단정하지 않고, θ 민감도와 거래비용을 함께 고려한 후보로 해석한다.

사건균형 필터는 수정 후 실제 작동했지만, baseline 대비 개선 폭은 제한적이었다. 따라서 최종 전략 요소로 채택하지 않고, HSI 상태분류를 보조적으로 설명하거나 조정하는 진단 후보로 둔다.

θ 민감도 실험은 HSI 상태분류가 특정 θ 값 하나에 과도하게 의존하는지 확인하기 위한 실험이다. θ는 예측 파라미터가 아니라 상태분류 민감도 조절값으로 해석한다.
8장: Macro companion layer

## 8. Macro companion layer: 금리·환율·GDP 보조 해석

Macro companion layer는 HSI를 대체하기 위한 전략이 아니다. HSI가 가격 기반 시장상태 해석 지표라면, macro companion은 금리, 환율, GDP를 이용해 HSI 판단을 보조적으로 해석하는 레이어이다.

12번 실험은 금리, 환율, GDP를 월별 보조신호로 정리한다. 금리 상승과 원/달러 상승이 동시에 나타나고, 금리-환율 이탈률이 높을 때 위험형 이탈로 해석한다. GDP 성장률은 공식 정책 목표가 아니라 내부 기준으로 사용하며, 성장 둔화 또는 성장 압력 구간을 판정하는 데 사용한다.

13번 실험은 HSI가 위험이라고 본 달에 macro companion도 위험 쪽을 같이 보고 있었는지 진단한다. 이 결과는 macro 보조값을 무조건 적용하지 않고, HSI와 macro 위험 신호가 겹치는 구간에서만 강하게 적용하기 위한 근거가 된다.

14번 실험은 HSI baseline 비중을 유지하되, macro 위험 신호가 겹치는 구간에서 위험자산 비중을 소폭 줄이는 macro overlay 후보이다. 이때 줄인 위험자산 비중은 국고채 3년과 단기채권PLUS로 나누어 이동시킨다. 금리 상승 구간에서는 채권도 손실을 볼 수 있으므로, 단기채권PLUS 쪽에 더 많이 배분하는 구조를 우선 검토한다.

정리하면, HSI는 주 판단이고 macro companion은 작은 보정이다.


## 9. 최종 후보 조합과 해석법

현재까지의 실험 결과를 기준으로 최종 후보 조합은 다음과 같이 해석한다.

| 구분 | 역할 | 판단 |
|---|---|---|
| EW | 단순 비교 기준 | benchmark로 유지 |
| HSI baseline | HSI 상태-비중 연결 기준선 | 최종 후보가 아니라 기준선 |
| λ=0.3 | 균형형 부분조정 후보 | 우선 검토 후보 |
| λ=0.1 | 방어형 저회전 후보 | 보조 후보 |
| event balance filter | 사건균형 보조 필터 | 진단 후보 |
| macro companion | 금리·환율·GDP 보조 레이어 | 후속 검토 후보 |

따라서 현재 결론은 HSI baseline 자체를 최종 전략으로 채택하는 것이 아니다. HSI 상태분류를 기준으로 하되, 비중 전환 속도를 λ로 조절하는 구조가 더 안정적인 후보로 나타났다. 사건균형지표와 macro companion은 HSI를 대체하지 않고, HSI 상태를 설명하거나 소폭 조정하는 보조 레이어로 남긴다.

최종 해석은 다음과 같이 해야 한다.

> HSI overlay 전략은 단순히 위험자산을 많이 사거나 적게 사는 전략이 아니다. HSI 상태분류를 통해 시장상태를 해석하고, λ 부분조정을 통해 비중 전환 속도를 낮추며, 사건균형지표와 macro companion을 보조 진단 레이어로 붙이는 구조이다. 현재 기준에서는 λ=0.3을 균형형 후보, λ=0.1을 방어형 후보로 두고 후속 검증한다.

1. 수익률 연율화

월간 수익률($r$)

$$\text{Annualized Return} = (1 + \text{average\_monthly\_return})^{\text{periods\_per\_year}} - 1$$

2. 변동성(표준편차) 연율화

변동성은 시간의 제곱근에 비례하여 커지는 성질이 있습니다. 

월간 변동성($\sigma_{monthly}$)을 연간 변동성($\sigma_{annual}$)으로 바꿀 때는 다음을 사용합니다.

$$\sigma_{annual} = \sigma_{monthly} \times \sqrt{\text{periods\_per\_year}}$$
